## Repository Preservation Principle

The primary purpose of the repository is preservation and transmission of knowledge.

Architecture exists to support knowledge, not to replace it.

Once publication quality has been achieved, preserving repository stability takes precedence over pursuing structural perfection.

A stable repository that readers can reliably access and reproduce is more valuable than a perfectly organized repository that remains unpublished.

Preserve first.

Release second.

Refactor in the next cycle.
